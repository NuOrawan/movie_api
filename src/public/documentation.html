myFlix Application

Documentation

myFlix Application is REST API that allow clients to access to information about different movies, directors, and genres via commonly used HTTP methods like GET, POST, PUT and DELETE. The application allows clients to update their information, for instance, their list of “Favorite Movies", usernames, passwords and emails.

Date of birth of director is in ISO 8601 format.

URL Endpoints and Request Methods

Get all movies

URL

/movies

Method

GET

Request Body Format

None

Response Body Format

A JSON Object holding data about all movies.

Get description, genre, director, image URL of a single movie by title.

URL

/movies/[title]

Method

GET

Request Body Format

None

Response Body Format

A JSON object holding data about a single movie containing ID, title, description, genre, director and image. Example :
[
id :1,
title : "Jurassic Park 2"
description : “is based on the 1990 novel written by Michael Crichton. “,
genre : {
name :“action”,
description :"fast-paced and include a lot of action like fight scenes, chase scenes."
}
director : {
name : “Steven Spielberg”,
dob : 1946-12-18,
deathyear :,
bio : “Spielberge was born in Ohio and grew up in Arizona. He move to California and studied film in college."
imageURL : "/img/jurassic_park_2.jpeg" }
]

Get data about a genre ‘s description by name.

URL

/movies/genre/[name]

Method

GET

Request Body Format

NONE

Response Body Format

A JSON Object holding about a genre. Example :
[
genre {
name : "science fiction" ,
description: "A genre characterized by stories involving conflicts between science and technology, human nature."
}
]

Get data about a director (bio, date of birth and death year) by name.

URL

/movies/directors/[name]

Method

GET

Request Body Format

NONE

Response Body Format

A JSON object holding data about a director. Example :
director : {
name : “Steven Spielberg”,
dob : 1946-12-18,
deathyear :,
bio : “Spielberge was born in Ohio and grew up in Arizona. He move to California and studied film in college."
}
]

Add new user.

URL

/users

Method

POST

Request Body Format

A JSON object holding data about a new user to be added. Example :
[
name : “Paula Gormez”,
username : “plgormez”,
password : “PL$2023mz”,
email : “pgormez@gmail.com”,
favoriteMovies : [“Babylon”]
]

Response Body Format

A JSON object holding data about a new user that was added including id. Example :
[
id :1,
name : “Paula Gormez”,
username : “plgormez”,
password : “PL$2023mz”,
email : “pgormez@gmail.com”,
favoriteMovies : [“Babylon”]
]

Update user info. by username.

URL

/users/[username]

Method

PUT

Request Body Format

NONE

Response Body Format

A text message indicating whether username has been updated successfully.

Add a movie to user’s list of favorites

URL

/users/[id]/[movieTitle]

Method

POST

Request Body Format

A JSON object holding data about a user’s favorite movies. Example :
[
favoriteMovies : [
“Hustle”,
“Top Gun Maverick”,
“Ambulance”
]

Response Body Format

A text message indicating whether a movie has been added successfully.

Remove a movie from a user’s list of favorite movies.

URL

users/[id]/[movieTitle]

Method

DELETE

Request Body Format

A JSON object holding data about a user’s movies to be removed. Example :
favoriteMovies : [
“Hustle”,
]

Response Body Format

A text message indicating whether a movie has been removed successfully.

Deregister/remove existing users.

URL

/users/[id]

Method

DELETE

Request Body Format

NONE

Response Body Format

A text message indicating whether user’s email has been removed successfully.
